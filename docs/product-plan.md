# 产品规划

## 一句话产品目标

基于《中国相术学》及其结构化知识库，打造一个面向中国及海外华人的微信小程序，让用户通过上传或拍摄面部照片，获得整体相学分析以及针对具体问题的方向性指引。

## 目标用户、核心场景、MVP

### 目标用户

- 对传统相学、命理、个人成长感兴趣的中国用户
- 海外华人，偏好中文服务，希望获得低门槛、轻量化的分析体验
- 当前有具体困惑，希望获得方向性建议的用户，例如感情、事业、财运、人际、健康习惯等
- 喜欢分享测试结果、愿意把结果转发给朋友的社交型用户

### 核心场景

- 用户上传一张正脸清晰照片，希望获得一次完整的整体相学分析
- 用户通过手机摄像头拍摄照片，并围绕一个具体问题寻求方向性指引，例如“最近适合换工作吗”
- 用户希望快速看到“面部哪些部位被观察到、对应什么解释、为什么会得出这个结论”
- 用户分析完后保存记录，后续带着不同问题反复回来使用
- 用户将分析结果分享给朋友，形成自然传播与裂变

### MVP

- 微信登录
- 上传照片或调用摄像头拍照
- 基础图片质量检测
  - 是否正脸
  - 是否清晰
  - 是否光线足够
  - 是否存在严重遮挡
  - 是否为多人合照
- 生成两类结果
  - 整体相学分析
  - 单个问题的方向性指引
- 结果页展示
  - 观察到的面部特征
  - 对应的相学解释
  - 面向用户的方向性建议
- 历史记录查看
- 风险提示与免责声明
  - 明确说明内容为相学文化解读与娱乐参考
  - 不替代医疗、法律、投资等专业意见

## 推荐技术栈

### 前端

- 微信小程序原生框架
- TypeScript
- 自定义组件体系

说明：
- MVP 阶段优先选择微信小程序原生能力，减少额外框架成本和兼容性问题
- 先把上传、拍照、结果展示、历史记录跑通，再考虑 UI 框架或复杂状态管理

### 后端

- `NestJS` 或 `Fastify + TypeScript`

说明：
- 如果希望项目结构更规整、便于多人协作，优先 `NestJS`
- 如果希望更轻量、接口响应更快，`Fastify + TypeScript` 也可行

### AI 与视觉能力

- 人脸检测与关键点识别
  - 国内优先可选腾讯云、阿里云、百度智能云的人脸关键点能力
  - 如后续考虑海外扩展，可预留 AWS Rekognition 或同类方案适配层
- 大语言模型
  - 用于把结构化特征、规则结果和用户问题组织成自然语言报告

说明：
- 不建议让模型直接裸看图并生成结论
- 更稳妥的做法是先完成“视觉特征提取”，再映射到相学知识模型，最后由大模型负责文案生成与表达控制

### 知识层与数据层

- PostgreSQL
- 可选 `pgvector` 作为后续知识检索增强能力

说明：
- 当前已经有结构化知识文件 `xiangxue_knowledge_complete_149p.json`
- 该知识库包含 `core_models`、`thematic_index` 和逐页原文，足以作为初版知识底座
- MVP 阶段可以先以规则映射和字段化知识为主，不必一开始就引入复杂向量检索

### 存储、缓存与任务系统

- 图片存储：腾讯云 COS
- 缓存：Redis
- 异步任务：BullMQ 或简单任务表机制

说明：
- 图片上传、分析任务、报告生成建议走异步流程，避免前端阻塞等待
- Redis 可以同时承担缓存与任务队列支持

### 部署与基础设施

- 腾讯云 CloudBase、CVM 或容器服务

说明：
- 如果主打微信生态，优先腾讯云会更顺手，尤其在对象存储、鉴权、域名备案和生态兼容性上更稳

### 合规与风控

- 小程序隐私协议
- 用户授权说明
- 图片存储与删除机制
- 审核敏感文案控制

说明：
- 结果文案应避免出现医疗、法律、投资、命运断言等高风险表述
- 产品定位应明确为相学文化内容、方向性建议和娱乐参考

## 10个开发任务

### 1. 明确产品边界与合规口径

- 明确产品是“相学文化解读 + 方向性建议”
- 不承诺预测结果
- 不提供医疗、法律、投资等专业判断
- 明确小程序审核文案、免责声明和隐私授权范围

### 2. 规范化整理结构化知识库

- 将 `xiangxue_knowledge_complete_149p.json` 中的内容按产品需要拆成可执行字段
- 重点整理以下几层
  - 五官
  - 三停
  - 十二宫
  - 主题索引
  - 逐页原文溯源
- 输出统一的知识映射标准，例如“部位 -> 特征 -> 相学含义 -> 适用问题 -> 说明边界”

### 3. 设计分析引擎链路

- 先定义从照片到结果的完整流程
- 推荐链路如下
  - 图片上传
  - 人脸质量校验
  - 面部区域识别
  - 特征抽取
  - 映射到相学规则与知识模型
  - 调用大模型生成自然语言报告
- 区分“整体分析”和“问题导向分析”两条生成路径

### 4. 设计数据模型与数据库表

- 用户表
- 图片表
- 分析任务表
- 特征提取结果表
- 相学规则表
- 问题模板表
- 报告结果表
- 历史记录表
- 审核日志或风控表

### 5. 搭建微信小程序基础流程

- 首页
- 拍照/上传页
- 分析中状态页
- 结果页
- 历史记录页
- 用户中心页

目标是先跑通从进入小程序到看到报告结果的最短闭环。

### 6. 搭建后端 API 能力

- 登录鉴权接口
- 图片上传接口
- 创建分析任务接口
- 查询分析结果接口
- 查询历史记录接口
- 预留后台管理接口

### 7. 接入人脸检测与图片质量控制

- 检测是否正脸
- 检测是否清晰
- 检测是否存在遮挡
- 检测是否多人脸
- 检测角度是否可用于分析

这一步的目标不是先做复杂分析，而是先保证输入质量。

### 8. 落地第一版相学知识映射与报告模板

- 基于已有 JSON，先做第一批高频规则
- 建议首批覆盖
  - 五官总纲
  - 三停
  - 十二宫中的事业、财帛、夫妻、命宫等高频主题
- 同时建立整体分析模板和问题导向模板

### 9. 建立结果生成、风控与解释机制

- 每份结果都尽量展示
  - 识别到的部位
  - 命中的特征
  - 对应的相学解释
  - 最终给用户的方向性建议
- 增加敏感词与高风险表达控制
- 保证结果不走极端结论，不输出强断言

### 10. 小范围内测与迭代优化

- 测试真实用户是否愿意上传照片
- 测试结果是否易懂、是否有复用价值
- 测试用户是否会继续发起第二个问题
- 根据反馈迭代拍照引导、结果解释、文案语气和推荐路径

## 风险点与容易失败的地方

### 1. 知识库虽完整，但不等于产品规则可直接用

- 当前 JSON 已经非常适合作为知识底座
- 但仍然需要进一步转成可执行规则，否则只能做检索展示，难以稳定生成分析结果

最容易失败的情况：
- 直接把整份 JSON 塞给模型生成结果
- 导致输出漂移、重复、无法解释、难以校验

### 2. 直接让模型“看图说话”会非常不稳定

- 如果跳过特征抽取与规则映射，让大模型直接根据图片输出相学判断，结果会极不稳定
- 同一张图在不同轮调用中可能出现不一致结论

最容易失败的情况：
- 结果像随机生成
- 用户无法理解“为什么这么说”
- 后续无法做版本迭代与质量控制

### 3. 图片输入质量会直接决定体验上限

- 模糊
- 侧脸
- 遮挡
- 滤镜过重
- 美颜过强
- 光线不足

最容易失败的情况：
- 分析结果失真
- 用户认为产品不准
- 投诉集中在“我明明拍了照，却没说到点上”

### 4. 合规与平台审核风险高

- 微信小程序对迷信、算命、命运承诺、财运断言等表述较敏感
- 如果文案不收敛，很容易在审核和后续运营阶段遇到问题

最容易失败的情况：
- 首页宣传语过度承诺
- 结果页出现命运定论
- 涉及健康、婚姻、投资等高风险判断

### 5. 结果缺少可解释性会削弱信任

- 用户不一定要求绝对准确，但会要求结果“像是有依据的”
- 如果结果页只给一句结论，没有部位观察与逻辑链条，用户很容易认为是随机生成

最容易失败的情况：
- 只有结论，没有证据
- 看起来像通用鸡汤
- 用户不愿意二次使用

### 6. “问题导向分析”比“整体分析”更容易失控

- 整体分析相对容易做成固定模板
- 一旦进入事业、感情、财运等具体问题，模型更容易输出过度具体、过度肯定的建议

最容易失败的情况：
- 出现“你应该辞职”“你不适合结婚”“这笔投资会亏”之类的高风险话术
- 用户把娱乐内容误解成决策依据

### 7. 海外华人并不只是中国版的自然延伸

- 海外用户在语言习惯、网络环境、支付方式、文化接受度上与国内用户不同
- 如果一开始同时兼顾中国和海外华人，MVP 会被拖复杂

最容易失败的情况：
- 首版同时做太多国际化适配
- 反而让最核心的拍照、分析、结果闭环不够稳

### 8. 过早做大而全，会拖慢验证速度

- 如果一开始就追求所有面部细节、所有主题、所有宫位、所有解释都上线，工程量会急剧膨胀

最容易失败的情况：
- 长时间停留在“知识整理”阶段
- 产品迟迟没有最小闭环
- 无法快速验证用户是否真的买单

### 风险控制建议

- MVP 先聚焦“整体分析 + 单问题方向建议”
- 首批只覆盖高频主题，不追求全量知识一次上线
- 坚持“视觉特征提取 -> 规则映射 -> 文案生成”的链路
- 强化免责声明、审核词表和结果解释性
- 先验证中国市场闭环，再考虑海外华人扩展
