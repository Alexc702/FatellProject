# FaceXiang v1 Identity & Memory Spec

## 1. 目标

本文件定义首版的：

- 身份策略
- 数据归属
- 记忆权限
- 纠错与删除闭环

目标是同时满足：

- `iOS first`
- `first aha` 低摩擦
- 7 天线程连续性

## 2. 决策摘要

v1 采用：

- `anonymous cloud-first`
- 设备端 `Keychain` 持久化匿名身份
- 服务端主存，客户端缓存
- 允许后续升级为 `Sign in with Apple`

不采用：

- 首次使用强制登录
- 纯本地记忆
- 无法删除的隐式长期存档

## 3. 身份生命周期

## 3.1 首次启动

1. App 生成 `installation_id`
2. 调用后端创建匿名用户
3. 返回：
   - `user_id`
   - `session_token`
4. 存入 `Keychain`

说明：

- 用户在 first session 无需先登录
- 但线程和记忆已经有稳定归属

## 3.2 后续启动

1. 从 `Keychain` 读取匿名身份
2. 拉取服务端主画像、线程和 recap
3. 本地缓存更新

## 3.3 升级登录

支持在未来任意时点升级绑定：

- `Sign in with Apple`

升级后：

- 继承原匿名账号全部记忆和线程
- 不新建第二个用户档案

## 3.4 删除账号

用户可在设置页执行：

- 删除图片
- 删除记忆
- 删除账号

删除后：

- 服务端清理用户主档
- 线程与 recap 清理
- 本地 token 作废

## 4. 数据归属原则

### 4.1 服务端主存

以下数据以服务端为准：

- `UserProfile`
- `MemoryItem`
- `GuidanceThread`
- `ThreadMessage`
- `DailyGuidance`
- `WeeklyRecap`

### 4.2 本地缓存

本地只做：

- 最近结果缓存
- 页面加载加速
- 短期离线容错

### 4.3 敏感图片

图片处理原则：

- 默认只保留必要时长
- 不把原图当长期关系资产
- 允许用户主动删除

## 5. 记忆分类

首版记忆按 5 类组织。

## 5.1 事实记忆

示例：

- 工作状态
- 感情状态
- 重要人物
- 当前目标

特点：

- 更偏客观
- 允许用户随时修改

## 5.2 tension 记忆

示例：

- 最怕失去什么
- 当前最大压力源
- 当前最在意哪段关系或决定

特点：

- 更偏动态
- 优先级高于普通事实记忆

## 5.3 关系记忆

示例：

- 重要人物名称/代称
- 与用户的关系类型
- 当前关系阶段

特点：

- 只保留最必要的信息
- 不默认保存过多第三方隐私

## 5.4 线程记忆

示例：

- 当前主线程
- 历史线程
- 上次建议
- 用户反馈

## 5.5 总结记忆

示例：

- 本周 recurring pattern
- 哪类建议有效
- 哪类表达让用户有共鸣

## 6. 记忆写入来源

每条记忆必须标记来源：

- `user_input`
- `user_correction`
- `ai_inference`
- `system_summary`

策略：

- `user_input` 和 `user_correction` 权重最高
- `ai_inference` 不得直接变成高置信长期事实
- `system_summary` 只用于 recap 和摘要

## 7. 记忆置信度规则

建议每条记忆包含：

- `confidence`
- `status`

状态建议：

- `active`
- `needs_confirmation`
- `suppressed`
- `deleted`

原则：

- AI 推断出的敏感记忆，默认 `needs_confirmation`
- 用户手动纠正后，原错误记忆转为 `suppressed`

## 8. Memory Review 产品闭环

首版必须提供最小版 `Memory Review`。

用户应能：

- 查看当前主画像
- 查看当前主 tension
- 查看重要人物
- 编辑关键记忆
- 标记“这条不对”
- 标记“不要再提”

说明：

- 这不是高级功能
- 这是防止关系感崩塌的底线

## 9. 记忆使用规则

AI 在使用记忆时必须遵守：

### 9.1 优先使用最近确认过的记忆

最近被用户确认或纠正过的记忆优先级最高。

### 9.2 敏感记忆只有在强相关时才提起

例如：

- 失恋
- 家庭矛盾
- 职业危机

如果当前问题不相关，不应强行引用。

### 9.3 用户压制过的记忆不可复活

被标记为：

- `suppressed`
- `deleted`

的记忆不得在文案中再次出现。

## 10. v1 身份与记忆相关页面

### 10.1 My Story

展示：

- 当前主 tension
- 当前主线程
- 最近 7 天摘要

### 10.2 Memory

展示：

- 关键人物
- 当前关注主题
- 已确认记忆
- 纠错入口

### 10.3 Settings

提供：

- 删除图片
- 删除记忆
- 删除账号

## 11. 验收标准

进入更深的留存开发前，至少满足：

- 匿名身份在重启后可恢复
- 线程和记忆不会因重启丢失
- 用户能纠正至少一条关键记忆
- 被压制记忆不会再次被引用
