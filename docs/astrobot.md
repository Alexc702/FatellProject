# AstroBot 竞品调研与对标计划

## 1. 结论先行

`AstroBot` 不是一个“面相识别”产品，而是一个以“会说话的虚拟神谕”为包装的多占卜服务聚合 App。  
它的核心不是单次深度分析，而是：

- 用强视觉形象和语音交互制造“神秘陪伴感”
- 用每日内容和多服务入口提高打开频次
- 用 `XP` / 内购解锁高级服务
- 用多语言和多平台覆盖更广用户

结合 `product-plan.md` 和 `marketanalysis.md`，如果我们要做“同样一款产品”，不建议直接复刻成“多玄学工具箱”，而应做成：

- 前台体验上像 `AstroBot`
- 底层能力上比 `AstroBot` 更强
- 核心差异点从“泛占卜聚合”升级为“东方相学 + AI 视觉分析 + 问题导向解读”

一句话建议：

`AstroBot` 值得抄的是产品壳、陪伴感、留存结构和轻付费机制，不值得抄的是过于分散的功能集合与较弱的专业壁垒。

## 2. 背景对齐

根据现有两份文档，你们当前项目的底层方向已经比较清晰：

- `product-plan.md`：当前核心能力是“上传/拍摄正脸照片 -> 视觉特征提取 -> 相学知识映射 -> 生成整体分析与问题导向建议”
- `marketanalysis.md`：如果优先做海外，更适合定位为 `self-discovery / relationship guidance / life direction` 产品，而不是直接强调“算命”

这意味着你们的真正优势是：

- 有相对成体系的东方相学知识库
- 有明确的图片分析主链路
- 已经意识到合规边界和现代化表达的重要性

而 `AstroBot` 的优势更多在：

- 用户包装层
- 入口丰富度
- 神秘感视觉与语音体验
- 每日消费场景

所以对标方式不该是“功能一比一复制”，而应该是“借它的消费产品结构，承载你们更强的分析引擎”。

## 3. AstroBot 公开信息整理

以下信息来自截至 `2026-03-19` 可公开访问的 App Store、Google Play 和 RoboBot Studio 官方站点。

### 3.1 基本信息

- 产品名：`AstroBot`
- 开发者：`RoboBot Studio`
- 定位：`Fortune Teller / Talking Oracle`
- 平台：
  - iOS
  - Android
  - Windows 10

### 3.2 核心卖点

从官方描述和商店文案看，`AstroBot` 的主卖点是：

- 每日运势
- 塔罗牌
- 符文
- 数秘术
- 中国占星术
- 玛雅占星术
- 凯尔特生肖
- 梦境解析
- 宝石含义
- 手相
- 咖啡杯占卜
- 可语音播报的“说话神谕”

它不是强调“最准”，而是强调：

- 内容很多
- 每天可打开
- 有一个人格化角色为你解读

### 3.3 交互结构

从截图和官方站点可反推，`AstroBot` 的交互结构大致是：

1. 用强视觉 IP 形象建立第一印象
2. 首页集中展示多个占卜服务入口
3. 用户进入某一服务后得到一段结果
4. 部分内容免费，部分内容需要 `XP` 或内购
5. 通过每日服务和提问式入口提升复访

这套结构的重点是“低门槛反复消费”，而不是“单次复杂任务完成”。

### 3.4 商业模式

公开文案显示它采用的是混合变现：

- 免费下载
- 广告
- App 内购买
- `XP` 经验值/点数解锁部分功能

其中 `XP` 机制很关键，因为它把“付费”包装成了“成长和解锁”，比直接硬付费更轻。

### 3.5 留存机制

`AstroBot` 的留存不是靠深报告，而是靠：

- 每日 horoscope / tarot / rune 等固定消费内容
- 问答式交互
- 多服务入口让用户总有下一项可点
- 语音与角色感增强陪伴属性

### 3.6 渠道和现状

截至 `2026-03-19` 可见的公开信号：

- iOS App Store 页面显示其长期在运营，但 iOS 版本历史里最近一次公开更新日期为 `2023-01-19`
- Google Play 页面显示：
  - 评分约 `3.6`
  - 评论约 `2.15 万`
  - 下载量 `100万+`
  - 页面显示最近更新日期为 `2025-11-27`
- 官方站点仍在展示并分发该产品

可见它不是爆款精品订阅产品，更像一个长尾但有规模的“玄学娱乐内容应用”。

## 4. AstroBot 值得借鉴的地方

### 4.1 强角色包装

`AstroBot` 最有效的不是某个单点功能，而是把一堆普通占卜功能装进了一个统一角色里。  
这会让产品从“工具”变成“陪伴式角色”。

对我们有价值的是：

- 做一个稳定的主视觉人格
- 统一语气、世界观和表达方式
- 让用户感觉自己是在“和一个懂东方玄学的顾问对话”

### 4.2 日常消费入口

`AstroBot` 很懂“让用户每天都能打开一下”。  
这一点正好能补你们当前方案里偏重“拍照后出报告”的单次使用缺口。

建议我们也必须加入日常轻内容：

- 今日能量
- 今日人际提醒
- 今日关系建议
- 今日行动提示

### 4.3 轻解锁而不是一次性硬付费

相比“上传一张图直接付费看完整报告”，`XP` / credits 更适合神秘学产品。

因为用户更容易接受：

- 免费看摘要
- 做任务得点数
- 点数解锁深度分析
- 想更快解锁再付费

### 4.4 多模块提升转化机会

`AstroBot` 的另一个优点是服务入口丰富。  
即便用户不做塔罗，也可能去看梦境、数秘或手相。

对我们的启发不是“也做十几个模块”，而是至少设计：

- 主模块：AI 面相解读
- 高频模块：每日 guidance
- 轻问答模块：单问题指引
- 增长模块：关系/配对分析

## 5. AstroBot 不该直接照搬的地方

### 5.1 功能太散，专业壁垒弱

`AstroBot` 的能力更像“占卜导航页”，覆盖很多，但每个模块未必足够深。  
这类产品的风险是：

- 用户容易觉得有趣，但不一定形成强信任
- 每个服务都浅，难以建立真正差异化

而你们已有知识库和视觉主链路，应该做“少而深”，而不是“多而浅”。

### 5.2 现代市场表达还不够克制

`AstroBot` 仍有明显的传统 fortune teller 表达。  
对于海外投放和苹果/谷歌审核，这类表达上限有限。

如果我们面向海外，应改成：

- `self-discovery`
- `relationship insight`
- `social impression`
- `life direction prompts`

避免强命运断言。

### 5.3 视觉很强，但可能过于老派

它的视觉 IP 记忆点强，但也带有较浓的旧时代神秘学 App 风格。  
如果我们直接模仿，会显得像老产品翻版。

建议方向：

- 保留“神秘感 + 角色化”
- 但视觉语言更现代、更高端、更克制
- 减少廉价“算命工具感”

## 6. 对当前项目的建议产品定义

## 6.1 推荐定位

建议定义为：

`一个基于东方相学知识与 AI 视觉分析的个人洞察产品，提供自拍解读、问题导向建议和每日陪伴式 guidance。`

如果做海外版，可表达为：

`AI Face Reading for Self-Discovery, Relationships and Life Direction`

### 6.2 推荐产品结构

建议做成“三层结构”：

#### 第一层：高频轻内容

- 今日 guidance
- 今日关系能量
- 今日行动提醒
- 简短可分享卡片

目标：

- 提高打开频次
- 降低首次使用门槛
- 建立日常消费习惯

#### 第二层：核心深度能力

- 自拍/上传照片
- 面部质量校验
- 面部区域识别
- 特征提取
- 规则映射
- 深度报告生成

输出结果建议包含：

- 观察到的部位与特征
- 对应相学解释
- 面向事业/关系/社交/决策的建议
- 风险边界说明

#### 第三层：复购和传播模块

- 单问题提问
- 关系配对分析
- 历史记录
- 分享卡片
- 连续签到 / 点数体系

## 7. 建议的功能规划

### Phase 1：先做 MVP 闭环

目标：验证“拍照解读是否有人愿意持续使用和付费”。

功能范围：

- 首页
- 今日 guidance
- 自拍/上传入口
- 图片质量校验
- 整体相学分析
- 单问题导向分析
- 结果页
- 历史记录
- 免责声明与隐私说明
- 点数体系 `credits`

这阶段不要做太多占卜分支，不建议同时上：

- 塔罗
- 梦境
- 数秘
- 咖啡杯占卜

否则会稀释产品主心骨。

### Phase 2：补留存和传播

目标：解决“做完一次报告后为什么回来”。

功能范围：

- 每日任务 / 连续签到
- 免费摘要 + 点数解锁完整内容
- 分享结果卡片
- 问答式 guidance
- 关系分析

### Phase 3：扩展多模块服务

只有在核心面相产品跑通后，再决定是否扩展相邻模块，例如：

- 手相
- 生日/生肖辅助解读
- 梦境解析
- 语音顾问模式

注意这里应遵循一个原则：

先扩“和当前知识模型相邻”的模块，再扩“纯新玄学模块”。

## 8. 推荐的信息架构

建议首页不是工具列表，而是“一个主角色 + 三个主入口”：

1. `Read My Face`
2. `Ask for Guidance`
3. `Today`

补充入口：

- `History`
- `Relationship`
- `Profile / Credits`

这样比 `AstroBot` 那种平铺很多功能入口更聚焦，更适合第一版建立认知。

## 9. 推荐商业模式

建议采用比 `AstroBot` 更清晰的混合变现：

- 免费：
  - 今日 guidance
  - 首次基础面相摘要
  - 每日签到得 credits
- 点数解锁：
  - 深度完整报告
  - 单问题分析
  - 关系分析
- 订阅：
  - 每日专属 guidance
  - 更多历史记录
  - 每周深入报告
  - 更高额度 credits

不建议第一版就重广告。  
如果目标是中高客单价或海外付费用户，广告会损伤信任感和质感。

## 10. 推荐技术方案

结合当前文档，建议技术方向如下：

### 10.1 客户端

- 海外优先：`iOS App`
- 华人私域承接：后续补 `微信小程序`

原因：

- `marketanalysis.md` 已明确更适合 `iOS App + Web funnel`
- `AstroBot` 本身也是 App 化产品，不是小程序
- 人脸图片上传、历史记录、内购、订阅在原生 App 中更顺

### 10.2 后端链路

建议沿用你们已有判断：

1. 上传图片
2. 图片质量检测
3. 人脸关键点与区域识别
4. 结构化特征抽取
5. 相学规则映射
6. LLM 负责自然语言解释与风格控制
7. 风控/免责声明后处理

### 10.3 数据模型补充

在现有表结构建议之外，新增以下产品化字段：

- `credit_wallet`
- `daily_checkin_log`
- `question_session`
- `share_card`
- `subscription_status`
- `content_unlock_log`

## 11. 视觉和品牌建议

建议不要直接复刻 `AstroBot` 的蓝紫色赛博女祭司形象，但可以借鉴其方法论：

- 有一个强记忆点主角色
- 角色在所有页面中保持统一
- 配合少量动态和语音增强陪伴感

更适合你们的视觉方向是：

- 东方相学来源
- 现代高质感 UI
- 神秘但不过度廉价
- 中英双语兼容

可以考虑的风格关键词：

- `modern mystic`
- `east asian ritual aesthetics`
- `premium guidance`

## 12. 风险与难点

### 12.1 最大风险不是技术，而是用户信任

用户真正会问的是：

- 为什么你说得像真的？
- 你的结论依据是什么？
- 照片会不会被滥用？

所以结果页必须解释：

- 看到了什么
- 为什么这么解释
- 哪些内容只是参考

### 12.2 审核和投放风控

无论 App Store、Google Play 还是广告平台，都要避免：

- 保证发财、婚姻成功、健康诊断
- 命运决定论
- 过度操控型付费文案

### 12.3 首版很容易做成“大而空”

这是最需要控制的地方。  
`AstroBot` 之所以能铺很多功能，是因为它是长年累积的内容产品。  
你们首版如果一开始就做十几个模块，大概率会失焦。

## 13. 90 天执行计划

### 第 1-2 周：产品定义与资料整理

- 明确首版只做“面相 + 今日 guidance + 单问题解读”
- 梳理知识库字段标准
- 定义报告结构
- 输出合规话术
- 确认品牌方向和主角色设定

产出：

- PRD
- 信息架构图
- 结果页内容结构
- 风控文案

### 第 3-5 周：核心能力开发

- 图片上传与质量检测
- 人脸关键点识别
- 特征抽取与规则映射
- 报告生成链路
- 用户、记录、credits 数据表

产出：

- 可跑通的端到端分析接口
- 一版可用报告样例

### 第 6-8 周：客户端 MVP

- 首页
- 上传/拍照
- 分析中状态页
- 结果页
- History
- Credits / 解锁页

产出：

- 可内测的 iOS MVP 或高保真原型

### 第 9-10 周：留存机制与支付

- 今日 guidance
- 签到
- credits 发放与消耗
- 首次免费摘要
- 完整报告解锁

产出：

- 基础转化漏斗

### 第 11-12 周：测试与上线准备

- 隐私协议
- 审核素材
- 埋点
- 分享页
- 首轮内测反馈修正

产出：

- 上线包
- 指标看板
- 首版增长素材

## 14. 建议的成功指标

首版不要盯太多指标，先看 6 个核心数：

- 首次上传完成率
- 报告生成成功率
- 免费摘要到付费解锁转化率
- 次日留存
- 7 日留存
- 人均每周打开次数

如果这些数不成立，再多占卜模块都没有意义。

## 15. 最终建议

如果目标是“做出同样一款产品”，最合理的落地方式不是复制一个 `AstroBot` 克隆版，而是做：

`AstroBot 的消费结构 + 你们自己的东方相学引擎`

推荐首版产品公式：

`AI 面相解读 + 每日 guidance + 单问题建议 + credits 解锁 + 强角色包装`

不推荐首版产品公式：

`面相 + 塔罗 + 梦境 + 数秘 + 手相 + 咖啡杯 + 多个玄学工具拼盘`

前者更有机会做出真正的产品心智，后者更容易做成看起来丰富但缺少核心壁垒的娱乐 App。
